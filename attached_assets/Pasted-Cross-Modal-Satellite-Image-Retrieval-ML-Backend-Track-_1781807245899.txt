competing. This is Challenge 11.

The core problem: Given a SAR (Synthetic Aperture Radar) satellite image,
retrieve the visually corresponding Optical satellite image of the same
geographic area — and vice versa. SAR and Optical images of the same location
look completely different (SAR uses radar backscatter, Optical uses RGB/NIR
reflectance), so this is a hard cross-modal matching problem that cannot be
solved with standard image similarity. We solve it using contrastive learning
to embed both modalities into a shared 512-dimensional vector space, then use
FAISS for fast nearest-neighbor retrieval.

My role: ML Engineer + Backend Developer (Maddy)
My tech stack: Python 3.11, PyTorch 2.x, FastAPI, FAISS, Supabase, Docker
Dataset: SEN1-2 (Sentinel-1 SAR paired with Sentinel-2 Optical, 4 seasons,
~180,000 pairs)
Deployment target: Single server or cloud VM (GPU preferred, must work on CPU
for demo)

TASK 2 — MODEL (DualEncoder + ProjectionHead)

Build the full model architecture:

ProjectionHead MLP:


Linear(in_dim, 512) → BatchNorm1d(512) → ReLU → Linear(512, 512)
Output is L2-normalized (unit sphere projection)
Supports configurable in_dim (default 2048 for ResNet50)


DualEncoder:


Two separate encoders: sar_encoder and optical_encoder
Each encoder = pretrained ResNet50 (ImageNet weights) with the final FC layer
replaced by ProjectionHead
forward(sar, optical) returns (sar_emb, optical_emb), both shape (B, 512)
Add encode_sar(x) and encode_optical(x) convenience methods for inference


Important: Load pretrained ResNet50 with weights=ResNet50_Weights.DEFAULT.
Freeze the first 2 layers (layer1, layer2) for the first 5 epochs, then unfreeze
for fine-tuning.


TASK 3 — LOSS FUNCTION (InfoNCE / NT-Xent)

Implement InfoNCELoss:


Takes (sar_emb, optical_emb) both shape (B, 512), already L2-normalized
Computes symmetric cross-modal contrastive loss:
loss = 0.5 * (sar_to_optical_loss + optical_to_sar_loss)
Uses temperature parameter τ (default=0.07)
Diagonal of the similarity matrix = positive pairs
All off-diagonal = negatives (in-batch negatives)
Return scalar loss + accuracy (fraction of correct top-1 retrievals in batch)


Also implement a hard-negative mining variant HardNegativeInfoNCELoss that
weights the hardest 25% of negatives by 2× in the loss.
